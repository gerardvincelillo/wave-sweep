# Implementation Checklist

Use this checklist to keep the repository organized and to make the next milestone visible.

## Documentation Baseline

- [x] Project vision is documented in `docs/PROJECT-VISION.md`.
- [x] A stack inventory exists in `docs/STACK-INVENTORY.md` or an equivalent repo-specific document.
- [x] This implementation checklist exists.
- [x] A repo README exists.

## Repo Organization

- [ ] The current milestone is explicitly named in docs or README.
- [ ] The repo scope and out-of-scope boundaries are written down clearly.
- [ ] Setup, run, and test commands are documented and still accurate.
- [ ] Architecture or module layout is documented at a level that is easy to revisit later.

## Delivery Readiness

- [x] Tests or a test directory are present.
- [ ] CI or workflow automation is present.
- [ ] The next feature slice is small enough to implement and verify in one focused pass.
- [ ] Known limitations and unfinished areas are documented honestly.

## Ongoing Habit

- [ ] Update the docs folder whenever the repo direction changes materially.
- [ ] Review this checklist before starting a new milestone.
- [ ] Close completed checklist items instead of leaving planning implicit.
